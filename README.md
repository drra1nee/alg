# Реализация алгоритмов сортировок, математический функций и стека

Проект представляет собой консольное приложение для демонстрации и тестирования алгоритмов и структур данных: математических функций, сортировок и стека

Запуск осуществляется из корневой директории командой:

```bash
python -m src.main
```

Тестирование стека и математических функций реализовано с использованием `pytest`:

```bash
pytest
```

---

### Структура проекта

- `src/main.py`: точка входа, интерактивное меню выбора функций
- `src/modules/`: модули с реализацией:
  - `math_functions.py`: итеративные и рекурсивные версии `factorial` и `fibo` с проверкой аргументов
  - `simple_sorts.py`: `bubble_sort`, `quick_sort`, `counting_sort`, реализация сортировок без вспомогательных функций
  - `complex_sort.py`: `radix_sort`, `bucket_sort`, `heap_sort` с вспомогательными функциями
  - `stack.py`: стек на основе списка с поддержкой `min()`, поддерживаются int и float
  - `stack_cli.py`: интерактивный интерфейс для работы со стеком
- `tests/`: модульные тесты:
  - `test_math.py`, `test_stack.py`: тесты математических функций и работы со стеком
  - `test_sorts/`:
    - `generators.py`: функции генерации массивов разных типов(случайные целый/дробные числа, массив в обратном порядке и др.)
    - `benchmark.py`: замер времени выполнения сортировки
    - `runner.py`: интерфейс для генерации тест-кейсов и запуска сравнительного тестирования сортировок

### Поддерживаемые функции

#### Математические функции
- `factorial(n)`, `factorial_recursive(n)` - на вход должны подаваться только `int` большие или равные нулю
- `fibo(n)`, `fibo_recursive(n)` - на вход должны подаваться только `int` большие или равные нулю, рекурсивная версия использует мемоизацию

#### Сортировки
Все функции сортировок принимают список и возвращают новый отсортированный список. Выполняется проверка типов элементов и корректности аргументов

- `bubble_sort(a)`, `quick_sort(a)` - работают с `int` и `float`
- `counting_sort(a)` - только для `int`, поддерживает отрицательные значения
- `radix_sort(a, base=10)` - только для `int`, поддерживает отрицательные числа и основание системы счисления большее или равное 2, использует вспомогательную функцию `_sort_radix`
- `heap_sort(a)` - работает с `int` и `float`, использует вспомогательную функцию `_heapify`
- `bucket_sort(a, buckets=None)` - для `int` и `float`, при значениях вне диапазона `[0, 1)` выполняется линейная нормализация, число блоков по умолчанию равно длине массива, использует `_insertion_sort` для сортировки внутри блока

#### Стек
- Реализован на основе list
- Однородность типов: после первого `push` тип фиксируется (`int` или `float`)
- Поддерживаемые операции:
  - `push` - добавляет элемент на вершину стека
  - `pop` - удаляет и возвращает элемент с вершины стека
  - `peek` - возвращает элемент с вершины стека без удаления
  - `min` - возвращает минимальный элемент в стеке за O(1) за счет вспомогательного стека
  - `len` - возвращает длину стека
  - `is_empty` - возвращает True, если стек пуст, иначе False
  - `clear` - полностью очищает стек
  - `show` - выводит строковое представление стека от вершины к дну


#### Тестирование сортировок
Через пункты меню 12 и 13 можно:
- сгенерировать массивы заданного типа (случайные целые/вещественные, почти отсортированные, с дубликатами, убывающие)
- протестировать выбранные сортировки на массиве
- получить таблицу с результатами: корректность (сравнение с `sorted()`) и время выполнения
